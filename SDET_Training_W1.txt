Gherkin
	- to bring req. in short and simple language, we use User-stories
	- gherkin is simple English language to describe BDD framework
	- User stories is converted to Gherkin files
	- Req. -> US -> Gherkin

	- gherkin provides step-by-step execution of a task for the following scenario
	- we are expected to use the keywords provided by gherkin language, as it is still a requirement doc.
	- it is case-sensitive
	- keywords
		+ Feature		- providing feature or module name
		+ Background		- combination of steps, which executes before each scenario
		+ Scenario		- task you are going to perform (Scenario = Test Case)
		+ Scenario Outline	- used to run the scenario with multiple data
		+ Given			- precondition step (pre-requisite)
		+ When			- the task you want to perform steps
		+ Then			- outcomes step
		+ And			- continuation of given, then step
		+ But			- negative step
		+ Example		- it stores the data for the execution in Scenario Outline
	- extension - .feature
	- aka feature file

	-Example of a Gherkin File
		Feature: Login Functionality
		Scenario: Login with valid credentials
		Given user is on login page
		When User enters username
		And User enters password
		And User clicks Login button
		Then User is logged in successfully
		And User is able to navigate to product page

	-Steps to integrate BDD with playwright
		1- npm init playwright@latest
		2- npm i -D @playwright/test playwright-bdd
		3- cucumber extension in vsc
		4- configure BDD(runner) in playwright.config.ts

			import {defineBdd} from 'playwright/bdd'
			
			defineBddConfig({
				feature: 'tests/features/**/*.feature',
				steps: ['tests/steps/**/*.ts],
				outputDir: '.tests'
			});

		5- update package.json	
			{
				"scripts":{
						"bdd:gen": "bddgen",
						"test": "playwright test",
						"bdd": "npm run bdd:gen && npm run test"
				}
			}
		6- Write feature file
		7- Run "npm run bdd"
